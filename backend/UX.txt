● Agora tenho toda a documentação da API. Aqui está o resumo completo:

  ★ Insight ─────────────────────────────────────
  A API do DataJud usa Elasticsearch por baixo — os endpoints terminam em
  _search e o body é uma query DSL do Elasticsearch (match, bool, must,
  search_after). Isso significa que temos acesso a queries poderosas: busca por
  CNJ, filtros combinados, paginação eficiente, etc. O campo numeroProcesso é o
  CNJ sem formatação (só números).

  Detalhe importante: o número CNJ 0702906-79.2026.8.07.0020 do PROJETO.md
  indica tribunal 8.07 = TJDFT, então o endpoint correto é
  api_publica_tjdft/_search.
  ─────────────────────────────────────────────────

  ---
  Agora tenho tudo para propor o design. Aqui estão 2 abordagens para a prova de
   conceito:

  Abordagem 1: Script Python simples (Recomendada)

  - Um único arquivo Python que recebe o CNJ, identifica o tribunal pelo código,
   faz o POST na API e exibe os andamentos formatados no terminal
  - Prós: Rápido de fazer, valida a API imediatamente, base para o sistema

         ↓

         ↓

  O que o sistema faz

  - Cadastro: Advogado cadastra CNJ. Sistema identifica tribunal, puxa dados
  iniciais do DataJud, salva no banco.
  - Monitoramento: Todo dia 7h, Celery consulta os ~1000 processos. Compara
  movimentos com o que já tem. Achou novo? Marca como pendente.
  - Tradução IA: Movimento novo passa pelo OpenAI — transforma "Juntada de
  petição de manifestação sobre o laudo pericial" em "O perito entregou o laudo
  e a outra parte se manifestou".
  - Notificação: WhatsApp pro cliente com o resumo traduzido.
  - Financeiro: Tabela simples — processo, cliente, valor honorário, custas,
  status (pago/pendente), gera link PIX.
  - Prazos: Extrai prazos dos movimentos (intimação, audiência), mostra lista de
   prazos próximos.

  Interface

  Web simples (React básico ou até HTML puro com FastAPI templates). Telas:
  - Lista de processos — tabela com filtro por status, cliente, tribunal
  - Detalhe do processo — movimentos, cliente vinculado, financeiro
  - Prazos — lista ordenada por data, com alerta de vencimento
  - Clientes — nome, telefone WhatsApp, processos vinculados
  - Financeiro — pendências, valores por cliente

  Banco (tabelas principais)

  - clientes (id, nome, telefone, email)
  - processos (id, cnj, tribunal, classe, cliente_id, status, data_ajuizamento)
  - movimentos (id, processo_id, codigo, nome, data_hora, notificado, resumo_ia)
  - financeiro (id, processo_id, tipo, valor, status, data_vencimento)
  - prazos (id, processo_id, tipo, data_limite, descricao, status)
